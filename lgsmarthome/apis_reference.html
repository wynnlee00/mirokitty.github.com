Messages API > Message a device

Message a device APIpost

정의

/v1/messages/device/<user-id>: 기기에게 메시지 전송 요청

설명

대상 기기에게 메시지를 전송합니다. 요청에 대한 응답은 MQTT 채널로 받습니다. 수신인에 대한 정보가 정확하지 않은 경우에는 400 에러를 응답 받습니다.

End Point

  • post

    https://api-qa.lglime.com/v1/messages/device/<device-id>

    TokyoDV
  • post

    https://api-qa.lglime.com/v1/messages/device/<device-id>

    USAOP

Parameter

  • Request
    Header

    {
	"x-session-id": "641dadccdc10a605",
	"x-service-id": "183b2989b6992c9e811d6f2e",
	"x-session-key": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJleHAiOjE0MzA2MTU1NjksImlzcyI6IiIsImp0aSI6InRlc3QifQ.HJVMrsZY463RdYPyzo2B76ZLetJDKiWGI_LSEnui-8M"
}

    Header Key Required Description
    x-service-id mandatory services API를 통해서 발급받은 인증 token. 서비스 생성시에 허가된 범위의 api에 대해서만 호출이 허용 됩니다.
    x-session-id mandatory LIME에서 발급된 id를 의미하며 세션키와 함께 사용됩니다.
    x-session-key mandatory 사용자/기기 등록이 완료 된 이후에 발급된 세션 정보 입니다.
    Body

    {
	"request": {
		"message": "Ping"
	}
}

    Body Key Type Required Description
    request object mandatory 기기에게 전달되는 메세지에 대한 정보. 포맷은 자유롭게 사용할 수 있습니다.
    message string mandatory 기기에게 전달되는 메시지 본문.
  • Response
    Body Key Type Required Description
    result_code string mandatory 요청에 대한 결과 코드로 각 코드에 대한 의미는 Link를 참조합니다.
    result object mandatory 제어 요청 성공 여부
