1Server List

LG SmartHome Platform는 'HTTPS' 및 'MQTT'라는 프로토콜(Protocol)을 사용하며, LG SmartHome Platform내 각 서버가 사용하는 End-Point(URL)의 도메인 이름(Domain Name)은 지역(국가)에 따라서 다를 수 있습니다. 사용자 계정(LG Account)을 생성할 경우에는 사용계정 생성시 선택한 지역(국가)과 LG SmartHome Plartform의 End-Point(URL) 지역(국가) 정보가 반드시 동일해야 합니다. 다를 경우에는 제품 관련 기능(등록, 제어 수정 및 삭제) 사용이 제한됩니다.

  • HTTPS : 개발자가 LG SmartHome Platform에서 제공하는 API에 대한 처리 요청을 했고, 이에 대한 결과를 바로 회신 및 확인 할 수 있는 경우(eg.사용자 정보 조회, 제품 정보 수정 등)를 의미합니다.
  • MQTT : LG SmartHome Platform와 개발자가 개발중인 스마트 홈 서비스(어플리케이션) 사이에 정보 전달용 채널로 사용되며, 경우는 다음과 같습니다.
    경우1. 개발자가 LG SmartHome Platform에서 제공하는 API에 대한 처리 요청을 했으나, 반영된 결과를 바로 확인 할 수 없는 경우(eg. 제품 제어 요청 등)
    경우2. LG SmartHome Platform이 스마트 홈 서비스(어플리케이션) 상태에 대한 정보를 바로 전송해야 하는 경우(eg. 메시지 알림 등)
Protocol 형상 Region URL
HTTP QA Tokyo https://api-qa.lglime.com
MQTT QA Tokyo mqtt://api-qa.lglime.com

2Method

LG SmartHome Platform은 HTTP에서 가장 일반적으로 사용되는 Method인 POST, GET, PUT 그리고 DELETE를 지원하고 있습니다. 이들은 각각 생성, 읽기, 갱신 및 삭제(CRUD)와 같은 동작을 LG SmartHome Platform에 처리 요청할 때 사용합니다. API Reference 페이지에는 사용하고자 하는 API 가 어떠한 Method 를 사용하고 있는지 명시 되어 있습니다. HTTP Method 에 대한 추가 설명은 다음의 URL 을 참고해 주시기 바랍니다.

* http://www.restapitutorial.com/lessons/httpmethods.html

※ 상기 URL의 문서 내용은 REST에 대한 이해를 돕기 위한 것이며, 일부 내용은 자사의 의견과 다를 수 있습니다.

header CRUD 예제
POST CREATE 사용자 등록, 제품 연결 등
GET READ 제품 상태 정보 확인, 사용자 정보 확인, 설정 내용 확인 등
PUT UPDATE 사용자 정보 수정, 제품 정보 수정 등
DELETE DELETE 사용자 탈퇴, 제품 정보 삭제 등

3API / Data Type

LG SmartHome Platform은 REST(Representational State Transfer) 형태의 API를 제공하고 있으며, API를 통해 주고 받는 데이터는 JSON(JavaScript Object Notation) 표준 포멧을 지원합니다.
API에 대한 처리 요청 시, 인터넷 미디어 타입을 “application/json”으로 지정해야 올바른 결과에 대한 회신을 받을 수 있으며, 상세 정의 내용은 아래 Wiki 페이지를 참고하시기 바랍니다.

  • 한글 : https://ko.wikipedia.org/wiki/REST
  • 영문 : https://en.wikipedia.org/wiki/Representational_state_transfer

※ 상기 URL의 문서 내용은 REST에 대한 이해를 돕기 위한 것이며, 일부 내용은 자사의 의견과 다를 수 있습니다.

4Common Headers

LG SmartHome Platform API에서 사용되는 공통 헤더(Common Header)로, 그 항목과 설명에 대한 내용은 아래 표를 참고하시기 바랍니다. ※ 동일한 명칭의 header 사용 시, API 요청에 따라 다른 값을 사용하는 경우가 있기 때문에 요구사항을 자세히 확인해 주세요.

header 명칭 설명
x-service-id 서비스 아이디 Service 생성시 발급되는 Service ID입니다.
① 대상 국가, Phase 및 제품에 대한 설정 정보를 포함합니다.
② My Services에서 Service 생성시 발급됩니다.
② My Services 상세 페이지에서 조회가 가능합니다.
x-session-id 세션 아이디 LG SmartHome Platform에서 사용되는 사용자 식별을 위한 세션 ID입니다.
① 사용자 세션 ID는 'Users > Register a Users' API를 통해 생성됩니다.
② 생성된 세션 ID는 'Users > Search for Users' API에서 조회가 가능합니다.

Edit a device's information 등의 API 에서는 이 세션 ID를 제품 식별용으로 사용합니다.
상세 내용은 아래 URL Parameter를 참고해 주세요.
x-session-key 세션 키 LG SmartHome Platform에서 사용되는 사용자 식별을 위한 세션 Key입니다.
① 사용자 세션 Key는 'Users > Register a Users' API를 통해 생성됩니다.
② 생성된 세션 Key는 'Users > Search for Users' API에서 조회가 가능합니다

x-session-id에 deviceId 값을 사용하는 경우 x-session-key에 제품 세션 Key를 사용해야 합니다.
① 제품 세션 Key는 제품 등록시 자동으로 생성 됩니다.
② 생성된 제품 세션 Key는 'Devices > View a device's profile' API에서 조회되며, DeviceSessionKey 의 데이터값을 참고해 주세요.

5URL Parameter

LG SmartHome Platform API에서 사용되는 공통 헤더(Common Header)로, 그 항목과 설명에 대한 내용은 아래 표를 참고하시기 바랍니다. ※ 동일한 명칭의 header 사용 시, API 요청에 따라 다른 값을 사용하는 경우가 있기 때문에 요구사항을 자세히 확인해 주세요.

header 명칭 설명
user-id 사용자 ID LG SmartHome Platform에서 사용자 식별을 위해 사용되는 x-session-id값과 user-id값은 동일합니다.
① 사용자 ID는 'Users > Register a user' API를 통해 생성됩니다.
② 생성된 사용자 ID는 'Users > Retrieve user information OR the user's devices' API 에서 조회가 가능합니다.
device-id 기기 ID LG SmartHome Platform에서 제품을 식별하기 위해 사용되는 제품 ID입니다.
① 제품 ID는 'Devices > View list of connected devices' API 에서 조회가 가능합니다.
② 제품 ID는 'Devices > Connect a device' API 로 제품 연결 시 사용됩니다.

6Device Type

SmartHome 서비스(앱)에서 사용할 제품군은 LG SmartHome Platform에서 Service 생성 시 개발자(사)가 선택하는 제품입니다. 이 제품군은 3자리 숫자로 구성되어 있고, Device Type가 사전에 정의되어 있습니다.
Device Type는 LG SmartHome Platform에서 제품 정보 또는 등록된 제품 목록을 요청할 경우 회신 되는 정보에 포함되어 있습니다. 개발자는 이 Device Type 만으로 그 제품이 어떤 제품을 의미하는지 확인이 가능합니다.

Device Type 목록과 정의는 아래 테이블을 참고해 주세요.

header 명칭
101 냉장고
201 세탁기
202 건조기
301 오븐
401 에어컨
501 로보킹

7Message ID

제품 제어에 대한 요청은 인터넷 및 제품 상태에 따라서 즉시 반영되지 않을 수 있습니다.
사용자는 제품 제어에 대한 결과를 회신 받기 전, 다른 제품을 제어하거나 정보를 변경과 같은 명령을 요청할 수 있습니다.
이 경우 처리 결과는 MQTT 채널을 통해 처리 결과를 받을 수 있고, 결과에 대한 요청이 무엇인지 구분하기 위해 LG SmartHome Platform은 Message ID 를 사용합니다.

  • ① Message ID 는 SmartHome Application에서 생성하는 값 입니다.
  • ② Message ID 는 Control a device API(/v1/devices/<device-id>)등에서 사용 됩니다.
    API 를 통해 제어 요청시 서버는 처리 요청을 받아 들였음에 대한 응답을 하게 됩니다. 이 때 이 응답은 가전 제품이 제어 요청을 반영했다는 것은 아닙니다.
    제품이 제어에 대한 처리를 완료한 경우 그 결과를 응답하게 되는데, 이 결과가 어떠한 요청에 대한 처리인지 구분이 어려울 수 있습니다.
    • 이 때 원래의 요청이 어떤 것 이었는지 확인 위해 사용합니다.
  • ③ Message ID 는 request 안에 선언되어야 합니다.

Request

{
   "request": {
        "cmd": {
            "Category": "Operation",
            "Property": "Mode",
            "Value": "1"
        },
        "msgId":"abcd1234"
    }
}

Response

{
    "result": {
        "cmd": {
            "Value": "1",
            "Property": "Mode",
            "Category": "Operation"
        },
        "userId": "680357bfa67dd56c",
        "virtualDevice": 1,
        "profileId": "airconditioner_1_kr",
        "deviceType": "airconditioner",
        "msgId": "abcd1234",
        "modelNm": "FQ250LC1P"
    },
    "result_code": "200"
}

8Code List

LG SmartHome Platform에서 개발자의 API 처리 요청에 대한 결과를 회신합니다.
• 정상 처리는 200으로 회신
• 비정상 처리는 200을 제외한 나머지 결과로 회신

※ API 처리 요청 결과에 대한 상세 값은 아래 테이블을 참고해 주세요.

code result description
200 QA 200 OK
400 LIME_ERROR_BAD_REQUEST 잘못된 요청
401 LIME_ERROR_UNAUTHENTICATED 인증 에러
403 LIME_NOT_AUTHORIZED 접근 권한 없음
404 LIME_ERROR_NO_DATA 데이터 없음
405 LIME_ERROR_NOT_ALLOWED_METHOD 지원하지 않는 Method에 대한 요청
410 LIME_ERROR_NONEXISTENT_DEVICE 존재하지 않는 디바이스
411 LIME_ERROR_NONEXISTENT_DEVICE_PROFILE 존재하지 않는 디바이스 프로파일
412 LIME_ERROR_NONEXISTENT_REGISTERED_DEVICE 등록되지 않은 기기
413 LIME_ERROR_NONEXISTENT_OBJECT 존재하지 않는 오브젝트
414 LIME_ERROR_MISSING_PARAMS 필수 입력항목이 누락됨
415 LIME_ERROR_UNAUTHORIZED_PARAMS 허용하지 않는 PARAMETER 들이 입력됨
416 LIME_ERROR_UNSUPPORTED_SERVICE 모델에서 지원하지 않는 서비스
422 LIME_ERROR_WRONG_CONTENT 명령안의 데이터가 제품 기능과 맞지 않음
423 LIME_ERROR_CONTROL_LOCKED 제어 권한 없음
424 LIME_ERROR_EXCUTION_FAILED 실행 불가
426 LIME_ERROR_FIRMWARE_UPDATE_FAILED 펌웨어 업데이트 에러
429 LIME_ERROR_TOO_MANY_REQUESTS 너무 많은 요청
430 LIME_ERROR_INVALID_CONTROL_REQUEST 유효하지 않은 제어 요청
